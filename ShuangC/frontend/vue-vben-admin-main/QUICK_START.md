## 📦 启动步骤

### 1. 启动 Mock 后端 (终端 1)
```bash
cd apps/backend-mock
pnpm dev
```
**输出:** Mock 服务运行在 `http://localhost:5320`

### 2. 启动前端开发服务器 (终端 2)
```bash
cd apps/web-antd
pnpm dev
```
**输出:** 前端服务运行在 `http://localhost:5173`

### 3. 访问系统
打开浏览器访问: `http://localhost:5173`

**默认登录账号:**
- 用户名: `vben`
- 密码: `123456`

## 🎯 功能测试步骤

### 测试流程 1: 单张图片推理
1. 登录系统
2. 点击左侧菜单 **"系统管理"** → **"路面缺陷检测推理"**
3. 在 **"选择模型"** 下拉框中选择 `yolo11s.pt (v1.0)`
4. 点击 **"上传单张图片"** 按钮
5. 选择一张本地图片
6. 点击 **"开始推理"** 按钮
7. 等待 2 秒（模拟推理）
8. 查看推理结果（显示检测框、置信度等）

### 测试流程 2: 从图片库选择
1. 点击 **"从图片库选择"** 按钮
2. 在弹窗中浏览图片文件夹：
   - 裂缝检测 (3张)
   - 坑洼检测 (2张)
   - 护栏损坏 (2张)
3. 点击文件夹查看图片
4. 选择多张图片（勾选复选框）
5. 点击 **"确定"**
6. 点击 **"开始推理"**
7. 查看推理结果
8. 使用 **"上一张"** / **"下一张"** 翻页
9. 点击 **"下载结果"** 下载压缩包

### 测试流程 3: 批量压缩包推理
1. 点击 **"上传文件夹压缩包"** 按钮
2. 选择一个 .zip 文件
3. 点击 **"开始推理"**
4. 系统自动轮询任务状态
5. 任务完成后显示所有结果
6. 翻页查看每张图片的推理结果

## 📊 Mock 数据说明

### 可用模型 (3个)
1. **yolo11s.pt** - 轻量级模型
2. **yolo11m.pt** - 中等精度模型
3. **yolo11l.pt** - 高精度模型

### 图片数据库 (8张)
- **裂缝检测/** - 3张图片
- **坑洼检测/** - 2张图片
- **护栏损坏/** - 2张图片
- **根目录** - 1张图片

### 模拟检测结果
每张图片会返回 2-3 个检测框：
- **类型**: crack (裂缝) / pothole (坑洼)
- **置信度**: 0.75 - 0.95
- **严重程度**: high / medium / low

## 🔧 配置后端真实 API

### 修改 API 地址
编辑文件: `apps/web-antd/.env.development`

```bash
# Mock 模式 (当前)
VITE_API_URL=http://localhost:5320

# 真实后端模式 (修改为你的后端地址)
VITE_API_URL=http://your-backend-server:8000/api
```

### 后端接口要求
参考文档: `DEFECT_DETECTION_README.md`

需要实现的接口：
1. `GET /system/models` - 获取模型列表
2. `GET /system/images/database` - 获取图片库
3. `POST /system/inference/single` - 单张推理
4. `POST /system/inference/batch` - 批量推理
5. `POST /system/inference/from-database` - 数据库推理
6. `GET /system/inference/task/:taskId` - 任务状态
7. `GET /system/inference/results/:taskId` - 推理结果
8. `GET /system/inference/download/:taskId` - 下载结果

## 🐛 常见问题

### 1. 菜单中没有"系统管理"选项
**原因:** 路由文件未自动加载

**解决:**
```bash
# 重启开发服务器
pnpm dev
```

### 2. Mock 数据不返回
**原因:** Mock 后端未启动

**解决:**
```bash
cd apps/backend-mock
pnpm dev
```

### 3. 推理一直 loading
**原因:** API 地址配置错误

**检查:**
1. 浏览器控制台是否有网络错误
2. Mock 后端是否正常运行
3. `.env.development` 中的 `VITE_API_URL` 是否正确

### 4. 图片上传后没反应
**原因:** 这是 Mock 模式，不会真正上传文件

**说明:** Mock 模式下会立即返回模拟的推理结果，真实后端需要实现文件上传逻辑
